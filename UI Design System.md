Portfolio Website — UI Design System v1.0
1. Core Philosophy

The website must feel handcrafted, premium, and timeless—not AI-generated, template-based, or overly animated.

The design should prioritize clarity over decoration. Every visual element must help visitors find information quickly.

2. Design Personality

Keywords:

Premium
Elegant
Minimal
Technical
Modern
Confident
Clean
Human-made
Timeless

Avoid:

Startup style
SaaS landing pages
Generic portfolio templates
Futuristic cyberpunk aesthetics
Glassmorphism overload
Excessive gradients
Neon colors
3. Visual Style

The interface should feel similar to a high-end product website rather than a personal portfolio.

Think:

Apple-quality spacing
Linear clarity
Stripe-level polish
GitHub simplicity

But it must remain unique and not imitate any brand.

4. Color Palette

Use a limited color palette.

Primary colors:

White
Off-white
Dark charcoal
Soft gray

Accent color:

Only one accent color across the entire website.

It should be used sparingly for:

Links
Active buttons
Highlights

Avoid colorful interfaces.

5. Typography

Typography should carry the design.

Large headings.

Comfortable reading.

Excellent spacing.

Never use decorative fonts.

6. Components

All components should belong to the same design language.

Buttons

Cards

Sections

Images

Badges

Statistics

Navigation

must all look like parts of one system.

7. Project Cards

Projects are the heart of the website.

Each project should look like a product showcase rather than a resume item.

Large screenshot.

Clear title.

Small description.

Technology tags.

Open Details button.

8. Motion

Animations should feel natural.

Purposeful.

Fast.

No flashy effects.

No unnecessary motion.

9. Responsive Design

Desktop experience comes first.

Tablet second.

Mobile must remain clean and easy to navigate.

Nothing should feel like a compressed desktop version.

10. Accessibility

High contrast.

Keyboard friendly.

Readable typography.

Proper spacing.

Accessible colors.

11. Performance

Every design decision must respect performance.

Avoid heavy effects.

Avoid unnecessary JavaScript.

Prioritize fast loading.

12. Final Principle

The visitor should never need to search for information.

Information must always be obvious.

Every section should answer one simple question:

Who are you?

What have you built?

Why should I trust you?

Where is the proof?

How can I contact you?
